# Run Logs (local)

One Markdown file per significant run: `YYYY-MM-DD-<slug>.md`
(e.g. `2026-07-04-linkedin-campaign-week-3.md`), based on
`runs/run-log-template.md`.

A run is significant when it involves 2+ agents or produces an external
artifact.

Purpose (Stage 1: repeatability and learning):

- reconstruct how an artifact was produced (workflow, agents, sources)
- capture what worked and what to change next time
- provide evidence for the approval checklist

Run logs are **local, per-user data** — gitignored except this README and the
template. They may reference personal knowledge, so they follow the same
privacy rule as `knowledge/personal/`.
